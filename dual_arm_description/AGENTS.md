# dual_arm_description

ROS2 ament_cmake package — URDF description for a dual-arm mobile robot with 6 wheels and two 7-DOF arms.

## Build & run

```bash
colcon build --symlink-install --packages-select dual_arm_description
source install/setup.bash
ros2 launch dual_arm_description display_rviz2.launch.py
```

Standalone joint control GUI (tkinter, not a launch file):
```bash
python3 joint_control_panel.py
```

## Package layout

- `urdf/dual_arm_1kg.urdf` — main robot model (auto-generated by SolidWorks exporter, mesh paths and limits fixed)
- `meshes/*.STL` — visual/collision meshes (base + 7 left arm + 7 right arm links, no wheel meshes)
- `launch/display_rviz2.launch.py` — robot_state_publisher + joint_state_publisher_gui + rviz2
- `joint_control_panel.py` — standalone ROS2 tkinter GUI publishing to `/joint_states`
- `urdf.rviz` — rviz config at package root (installed to share dir via CMakeLists.txt)
- `config/joint_names_dual_arm_description.yaml` — placeholder, effectively empty

## Robot structure

- **base_link** — chassis
- **Left arm**: laxis1_joint – laxis7_joint (all `type="revolute"`, 7-DOF)
- **Right arm**: raxis1_joint – raxis7_joint (all `type="revolute"`, 7-DOF)
- **Wheels**: fl_wheel, fr_wheel, ml_wheel, mr_wheel, rl_wheel, rr_wheel — defined in URDF but joints are missing or non-functional; no wheel meshes exist

## Key facts

- Package is purely ROS2 / ament_cmake. No ROS1 (catkin) files.
- Dependencies: rclpy, sensor_msgs, robot_state_publisher, joint_state_publisher_gui, rviz2, gazebo_ros, xacro.
- Build artifacts (`build/`, `install/`, `log/`) live at the colcon workspace root, not in this directory.
- `notes.md` (in Chinese) documents the ROS1→ROS2 migration issues and fixes applied.
- The `urdf.rviz` file must be installed to the share directory (already handled in CMakeLists.txt) or rviz2 falls back to `map` frame and fails.
- All 14 arm joints have limits: lower="-3.14" upper="3.14" effort="20" velocity="2.0" (±180°).
- Mesh paths use `package://dual_arm_description/meshes/` — matches the ROS2 package name.
- Visual/collision origins are set to match inertial origins to compensate for SolidWorks mesh geometry offset.
